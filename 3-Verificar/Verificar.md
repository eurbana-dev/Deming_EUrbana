# Fase V — Verificar

Esta fase corresponde a la **validación y verificación** de resultados.  
Se asegura que los entregables cumplen con los requisitos establecidos en la fase de planificación.

## Contenido

- Evidencias de pruebas unitarias y funcionales.  
- Comparación con requisitos del documento de especificación.  
- Validación de datasets y procesos ETL.

## Propósito

- Verificar que la API responde en tiempo y forma (< 500 ms).  
- Validar que los datos sembrados a través del ETL y el Seeder cumplen con el modelo esperado.  
- Revisar métricas de rendimiento, disponibilidad y seguridad.  
- Confirmar trazabilidad entre requisitos → implementación → pruebas.
